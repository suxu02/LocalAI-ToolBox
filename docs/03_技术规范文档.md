# ⚙️ LocalAI-ToolBox 技术规范文档 (V1.0)

## 1. 技术栈选型标准（决策依据）

基于“单人开发、快速落地、纯前端部署”的核心约束，我们选择**“快、轻、稳”**的技术组合：

| 技术层 | 选型方案 | 核心理由 | 备注 |
| :--- | :--- | :--- | :--- |
| **前端框架** | **Vue 3 (Composition API)** | 上手快，生态完善，Composition API 非常适合逻辑复用（如AI调用逻辑）。 | 配合 `<script setup>` 语法糖，代码量更少。 |
| **构建工具** | **Vite** | 秒级启动，热更新极快，极大提升开发体验。 | 比 Webpack 快 10 倍以上。 |
| **UI 组件库** | **Element Plus** | 国内最成熟的 Vue 3 组件库，文档全，适合后台/工具类产品。 | 直接复用表单、弹窗、布局组件，无需手写 CSS。 |
| **本地存储** | **Dexie.js** | IndexedDB 的封装库，提供类 SQL 的语法，支持 Promise。 | 比原生 IndexedDB 代码量减少 80%，更易维护。 |
| **文档解析** | **pdf.js** (PDF) <br> **mammoth.js** (Word) | 前端解析标准库，成熟稳定。 | 必须使用 ES Module 版本。 |
| **状态管理** | **Pinia** | Vue 3 官方推荐，轻量级，支持持久化插件。 | 管理用户配置、当前会话状态。 |
| **代码规范** | **ESLint + Prettier** | 自动格式化代码，统一风格，减少格式化噪音。 | 强制保存时自动格式化。 |

## 3. 数据库设计规范

使用 Dexie.js 定义数据库 Schema，确保数据流转有据可依。

**数据库名**：`LocalAIToolboxDB`
**版本号**：1

### 表结构设计：

1.  **`settings` (配置表)**
    *   `id` (String, 主键): 唯一标识 (如 'api_key')
    *   `value` (String/JSON): 配置值
    *   `updatedAt` (Number): 时间戳

2.  **`templates` (模板表)**
    *   `id` (Number, 自增主键)
    *   `title` (String): 模板标题
    *   `content` (String): 模板内容 (含变量)
    *   `variables` (Array): 提取出的变量列表 ['主题', '时间']
    *   `params` (Object): 参数配置 `{ temperature: 0.7 }`
    *   `createdAt` (Number): 创建时间

3.  **`knowledge` (知识库表)**
    *   `id` (Number, 自增主键)
    *   `name` (String): 文件名
    *   `content` (String): 提取的纯文本内容
    *   `wordCount` (Number): 字数统计
    *   `createdAt` (Number): 上传时间

4.  **`history` (历史记录表)**
    *   `id` (Number, 自增主键)
    *   `title` (String): 自动生成的标题
    *   `messages` (Array): 对话消息数组 `[{role: 'user', content: '...'}, ...]`
    *   `tags` (Array): 标签数组 `['公文']`
    *   `type` (String): 来源类型
    *   `createdAt` (Number): 创建时间

## 4. Git 版本控制规范

严格执行 Git 规范，让你的开发历史清晰可溯，方便回滚和排错。

### 4.1 分支管理策略
单人开发采用简化版 Git Flow：

*   **`main` 分支**：生产分支。**严禁直接提交代码**。只能通过合并 `develop` 分支更新。此分支代码必须保证是可运行、已测试的稳定版本。
*   **`develop` 分支**：开发主分支。日常开发基于此分支拉取功能分支。
*   **`feature/*` 分支**：功能开发分支。
    *   命名规则：`feature/模块名-功能名` (例：`feature/prompt-template-crud`)
    *   开发完成后，合并回 `develop` 分支并删除。

### 4.2 Commit Message 提交规范
每次提交必须写清楚“做了什么”，严禁提交诸如 "update"、"fix" 等无意义信息。

**格式**：`<type>(<scope>): <subject>`

*   **type (必填)**：
    *   `feat`: 新功能
    *   `fix`: 修复 Bug
    *   `docs`: 文档更新
    *   `style`: 代码格式调整 (不影响功能)
    *   `refactor`: 重构代码
    *   `perf`: 性能优化
    *   `test`: 增加测试
*   **scope (选填)**：影响范围 (如 prompt, knowledge, db)
*   **subject (必填)**：简短描述 (中文/英文均可，建议中文)

**示例**：
*   `feat(prompt): 新增模板变量自动识别功能`
*   `fix(knowledge): 修复 PDF 解析中文乱码问题`
*   `refactor(db): 优化 IndexedDB 批量写入性能`

### 4.3 .gitignore 配置
必须忽略以下文件，避免敏感信息泄露和环境污染：

```text
node_modules/
dist/
.DS_Store
*.local
.env
.env.*
!.env.example
```

## 5. 编码核心规范 (AI 辅助开发必读)

为了保证 AI 生成的代码可用、可维护，设定以下规范：

1.  **组件化原则**：
    *   一个 Vue 文件 < 300 行。超过必须拆分为子组件。
    *   逻辑代码 > 50 行，必须抽离到 `composables` 或 `utils`。

2.  **错误处理规范**：
    *   所有异步操作（API调用、数据库操作）必须使用 `try...catch...` 包裹。
    *   捕获到的错误，必须使用 `ElMessage.error()` 提示用户，严禁 `console.log` 了事。

3.  **注释规范**：
    *   复杂逻辑（如 RAG 检索算法、流式输出解析）必须注释。
    *   函数头部注释功能、入参、出参。
