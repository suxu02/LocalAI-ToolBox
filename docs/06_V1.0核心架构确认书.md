**## 2. 核心数据架构（严禁随意改动）**



**### 数据库 Schema**



**| 表名 | 主键 | 主要字段 | 说明 |**

**|------|------|----------|------|**

**| \*\*settings\*\* | id | id, value, updatedAt | 存储 API 配置 |**

**| \*\*templates\*\* | ++id | id, title, content, variables, params, category, createdAt, updatedAt | 提示词模板 |**

**| \*\*knowledge\*\* | ++id | id, name, content, wordCount, createdAt | 知识库文档 |**

**| \*\*history\*\* | ++id | id, title, type, messages, tags, relatedId, createdAt | 对话历史记录 |**



**\*\*标注：\*\* V1.1 新增功能时，只能新增字段或新增表，严禁修改已有字段的数据类型，严禁删除原有字段。**



**### 关键数据流**



**\*\*用户输入 -> AI 请求 -> 结果保存\*\* 闭环路径：**



**1. \*\*用户输入\*\*：**

&#x20;  **- PromptLab：用户填写模板变量，点击生成**

&#x20;  **- Knowledge：用户选择文档，输入问题**



**2. \*\*AI 请求\*\*：**

&#x20;  **- 调用 `useAI().sendMessage()` 函数**

&#x20;  **- 构建请求参数（messages, temperature, model）**

&#x20;  **- 流式获取 AI 响应**

&#x20;  **- 实时更新 UI 显示**



**3. \*\*结果保存\*\*：**

&#x20;  **- AI 响应完成后，触发 `onComplete` 回调**

&#x20;  **- 调用保存逻辑，将对话记录写入 `history` 表**

&#x20;  **- 包含完整的消息数组、类型标记、关联文档 ID 等信息**

**### Composables 接口契约**



**#### useAI.js**



**\*\*对外暴露函数：\*\***

**- `sendMessage(messages, options)`：发送消息到 AI**

&#x20; **- \*\*参数\*\*：**

&#x20;   **- `messages`：消息数组，格式为 `\[{ role: 'user', content: '...' }]`**

&#x20;   **- `options`：配置对象，包含：**

&#x20;     **- `temperature`：温度参数，默认 0.7**

&#x20;     **- `model`：模型名称，默认 'deepseek-chat'**

&#x20;     **- `onChunk`：流式数据回调，参数 `(chunk, accumulated)`**

&#x20;     **- `onComplete`：完成回调，参数 `(content)`**

&#x20;     **- `onError`：错误回调，参数 `(error)`**

&#x20; **- \*\*返回值\*\*：完整的 AI 响应内容**



**- `abort()`：中断 AI 请求**

&#x20; **- \*\*参数\*\*：无**

&#x20; **- \*\*返回值\*\*：无**



**\*\*标注：\*\* V1.1 只能扩展参数，不能改变原有参数顺序或删除参数。**

**## 4. V1.1 开发准入原则（红线）**



**1. \*\*数据库兼容性\*\*：**

&#x20;  **- 严禁修改 IndexedDB 的版本号**

&#x20;  **- 严禁修改已有表的字段数据类型**

&#x20;  **- 严禁删除已有表的字段**

&#x20;  **- 新增功能只能通过新增字段或新增表实现**



**2. \*\*核心接口稳定性\*\*：**

&#x20;  **- 严禁改变 `useAI.sendMessage()` 的核心参数结构**

&#x20;  **- 严禁修改 `onChunk`、`onComplete`、`onError` 回调的调用时机和参数**

&#x20;  **- 新增参数必须作为可选参数，保持向后兼容**



**3. \*\*数据流完整性\*\*：**

&#x20;  **- 保持 "用户输入 -> AI 请求 -> 结果保存" 的完整闭环**

&#x20;  **- 确保所有对话记录都能正确保存到 `history` 表**

&#x20;  **- 保持历史记录数据结构的一致性**



**4. \*\*状态管理可靠性\*\*：**

&#x20;  **- 严禁修改 `useSettingsStore` 的核心状态结构**

&#x20;  **- 确保 API Key 和 Base URL 的持久化机制正常工作**

&#x20;  **- 保持应用初始化时的设置加载流程**



**5. \*\*模块边界清晰\*\*：**

&#x20;  **- 各模块职责明确，避免功能交叉**

&#x20;  **- 新增功能应遵循现有模块划分**

&#x20;  **- 保持代码结构的一致性和可维护性**
