# 📦 前端数据结构规范 V1.1 (替代传统接口文档)

## 1. 外部接口对接规范

本项目核心依赖 DeepSeek API。我们需要定义前端如何发起请求。

### 1.1 基础配置
*   **Base URL**: `https://api.deepseek.com` (或用户自定义代理地址)
*   **认证方式**: `Authorization: Bearer <API_KEY>`
*   **核心端点**: `/v1/chat/completions`

### 1.2 请求体结构
前端发送给 AI 的数据结构。

```json
{
  "model": "deepseek-chat", // 默认模型
  "messages": [
    { "role": "system", "content": "你是一个专业的行政助手..." }, // 系统预设
    { "role": "user", "content": "帮我写一份通知..." } // 用户提问
  ],
  "stream": true, // 必须开启流式输出
  "temperature": 0.7 // 创造性参数 (0-2)
}
```

### 1.3 响应体结构
前端接收的流式数据块格式 (SSE)。

```json
{
  "id": "chatcmpl-xxxx",
  "choices": [
    {
      "delta": {
        "content": "通" // 每次返回的增量子段
      },
      "finish_reason": null
    }
  ]
}
```

## 2. 本地存储数据模型

使用 Dexie.js 封装 IndexedDB。以下是核心表结构设计。

### 2.1 配置表
存储用户的配置信息，键值对形式。

| 字段名 | 类型 | 说明 | 示例值 |
| :--- | :--- | :--- | :--- |
| `id` | String | 主键 | `api_key` / `base_url` |
| `value` | String/JSON | 配置值 | `sk-xxxxxxxx...` |
| `updatedAt` | Number | 更新时间戳 | `1715123456789` |

**关键数据**：
*   `api_key`: 用户的密钥。
*   `default_params`: 全局默认模型参数 (JSON字符串)。

### 2.2 模板表
存储提示词模板。

| 字段名 | 类型 | 说明 | 约束 |
| :--- | :--- | :--- | :--- |
| `id` | Number | 主键 | 自增 |
| `title` | String | 模板标题 | 必填，最大50字 |
| `content` | String | 模板内容 | 必填，含 `{{变量}}` |
| `variables` | Array<String> | 提取的变量列表 | `["主题", "时间"]` |
| `params` | Object | 模板独立参数 | `{ "temperature": 0.2 }` |
| `category` | String | 分类标签 | "公文类" |
| `createdAt` | Number | 创建时间戳 | 索引 |
| `updatedAt` | Number | 更新时间戳 | - |

### 2.3 知识库表 **【V1.1 重大变更】**
存储解析后的文档数据，支持全文与分块双模式。

| 字段名 | 类型 | 说明 | V1.1 变化 |
| :--- | :--- | :--- | :--- |
| `id` | Number | 主键 | 自增 |
| `name` | String | 文件名 | "员工手册.pdf" |
| `content` | String | 解析后的纯文本 | **保留**，全文模式使用此字段。 |
| `mode` | String | 检索模式 | **【新增】** `'full'` (全文) 或 `'chunk'` (智能)。默认 `'full'`。 |
| `chunks` | Array<Chunk> | 分块数据 | **【新增】** 仅当 `mode='chunk'` 时有值。见下方 `Chunk` 结构。 |
| `wordCount` | Number | 字数统计 | 用于计费提示。 |
| `createdAt` | Number | 上传时间戳 | 索引。 |

**子结构：Chunk (分块项)** **【V1.1 新增】**
```typescript
interface Chunk {
  id: string; // 分块唯一标识 (如 uuid)
  text: string; // 分块文本内容
  startIdx: number; // 在原文中的起始字符索引
  endIdx: number; // 在原文中的结束字符索引
  // BM25 索引数据可选存入此处，或运行时计算。建议仅存储文本，运行时计算词频以减少存储空间。
}
```

**兼容性说明**：
*   V1.0 旧数据默认 `mode` 为 `'full'`，无 `chunks` 字段，功能保持不变。

### 2.4 历史记录表
存储对话历史。

| 字段名 | 类型 | 说明 | 约束 |
| :--- | :--- | :--- | :--- |
| `id` | Number | 主键 | 自增 |
| `title` | String | 对话标题 | 自动截取首句前10字 |
| `type` | String | 来源类型 | "prompt" (工坊) / "knowledge" (知识库) |
| `messages` | Array<Message> | 对话消息列表 | 见下方 `Message` 结构 |
| `tags` | Array<String> | 标签列表 | `["公文", "已发送"]` |
| `relatedId` | Number | 关联ID | 若是知识库对话，关联 knowledge 表 ID |
| `createdAt` | Number | 创建时间戳 | 索引 |

**子结构：Message (消息项)**
```typescript
interface Message {
  id: string; // 唯一标识 (用于前端渲染key)
  role: 'user' | 'assistant' | 'system';
  content: string; // 消息内容
  timestamp: number; // 发送时间
  citations?: Citation[]; // 引用来源 (仅知识库回答有)
}

interface Citation {
  docId: number; // 关联文档ID
  docName: string; // 文档名
  snippet: string; // 引用片段文本 (在智能模式下，此处为具体 Chunk 的文本)
}
```

## 3. 运行时数据流转规范

定义组件间通信的数据格式。

### 3.1 RAG 检索上下文 **【V1.1 更新】**
当知识库提问时，前端拼接的 Prompt 结构根据 `mode` 动态生成：

**全文模式:**
```text
你是一个智能助手，请根据以下背景信息回答用户的问题。
【背景信息】
{knowledge.content} // 直接填入完整文档内容
【用户问题】
{question}
```

**智能模式:** **【V1.1 新增】**
```text
你是一个智能助手，请根据以下背景信息回答用户的问题。
【背景信息】
1. {chunk1.text}
2. {chunk2.text}
3. {chunk3.text}
// 仅填入 BM25 算法检索出的 Top 3 相关分块
【用户问题】
{question}
```

### 3.2 模板动态表单
模板解析后生成的表单配置：

```typescript
interface FormField {
  key: string; // 变量名 "活动主题"
  label: string; // 显示标签 "活动主题"
  type: 'input' | 'textarea'; // 输入类型
  default?: string; // 默认值
  required: boolean; // 是否必填
}
```
